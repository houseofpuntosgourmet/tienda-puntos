## Pages & Features

### 1. **Login Page** (Entry Point)
**Route:** `/` (unauthenticated)
**Endpoint:** POST `/api/auth/admin/login`

**UI Elements:**
- Email input field
- Password input field
- Submit button (disabled while loading)
- Error message display (red box)
- Centered card layout (Tailwind)

**Flow:**
```
Email + Password → POST /api/auth/admin/login → 
  → Save token + usuario to localStorage → 
  → Redirect to Dashboard
```

### 2. **Dashboard** (Main Hub)
**Route:** `/dashboard` (after login)
**Endpoints:** 
- GET `/api/clientes` (fetch all clients)
- GET `/api/premios` (fetch all prizes)

**Sections:**

#### 2a. Stats Card
- Total Clientes count

#### 2b. Create Cliente Form
**Button:** "Create Cliente"
**Fields:**
- nombre (text)
- whatsapp (phone)
- dni (ID number)

**Action:** POST `/api/clientes`

#### 2c. Clientes Table
**Columns:**

- Fecha Registro
- Estado
- Premios Disponibles (expandable)
**Features:**
- Expandable rows showing eligible prizes
- Premio eligibility: `puntosActuales >= puntosRequeridos`

---

### 3. **Asignar Puntos** (Assign Points to Cliente)
**Route:** `/asignar-puntos`
**Endpoints:**
- GET `/api/clientes` (search)
- GET `/api/reglas/activa` (fetch active rule)
- POST `/api/transacciones` (create transaction)

**Workflow:**
1. Search/Select Cliente (via ClienteSearch component)
2. Input Monto (purchase amount in pesos)
3. System calculates: `puntos = monto / factor` (using active rule)
4. Display calculated puntos
5. Confirm & Submit → POST `/api/transacciones`

**Validation:**
- Cliente must be selected
- Monto must be > 0 (number)
- Monto cannot be just "."

---

### 4. **Reglas de Puntos** (Point Rules CRUD)
**Route:** `/reglas`
**Endpoints:**
- GET `/api/reglas` (list)
- POST `/api/reglas` (create)
- PUT `/api/reglas/:id` (update)
- DELETE `/api/reglas/:id` (delete)

**Fields:**
- nombre (e.g., "Standard Rule")
- factor (e.g., 900 = 900 pesos per 1 punto)
- activo (toggle)
- vigencia (start/end dates) - optional

**Note:** Only 1 rule should be `activo: true` at a time (used for punto calculations)

---

### 5. **Premios** (Prizes/Rewards CRUD)
**Route:** `/premios`
**Endpoints:**
- GET `/api/premios` (list)
- POST `/api/premios` (create)
- PUT `/api/premios/:id` (update)
- DELETE `/api/premios/:id` (delete)

**Fields:**
- nombre (prize name)
- descripcion (description)
- puntosRequeridos (points needed to redeem)
- vigencia (optional validity period)
- activo (toggle)

**UI:** 
- Table with edit/delete actions
- PremioForm component for create/edit modal

---

### 6. **Canjes Pendientes** (Pending Redemptions)
**Route:** `/canjes`
**Endpoints:**
- GET `/api/canjes?estado=pendiente` (fetch pending)
- POST `/api/canjes/:id/completar` (mark as done)

**Columns:**
- Cliente nombre
- Premio name
- Puntos requeridos
- Fecha solicitud (request date)

- Action: "Marcar como completado" button

**Status Flow:**
- `pendiente` → admin clicks "Completar" → `completado`

## Auth Flow

```
1. Check localStorage on mount (admin_token, admin_usuario)
   ↓
2. If exists → setToken + navigate to dashboard
   ↓
3. If not → show Login page
   ↓
4. On login → POST /api/auth/admin/login
   ↓
5. Save token + usuario to localStorage
   ↓
6. All API calls include: Authorization: Bearer {token}
   ↓
7. Logout → clear localStorage + navigate to login
```
